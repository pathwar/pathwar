How HTTP cookies work

A cookie is typically set by the server by sending a Set-Cookie header to the client (per cookie). A response that set cookies looks like this:

GET / HTTP/1.1
Host: foo.com

HTTP/1.1 200 OK
Set-Cookie: bar=42; Path=/; Domain=.foo.com
Set-Cookie: session=12345; HTTPOnly
...

_('Cookies are automatically sent by the client with the header 'Cookie' containing all the valid cookies for the current domain, separated by a semicolon. (You will have the occasion to study this behaviour later in some levels :)). Using the above example, any subsequent request to foo.com will include the two cookies set by the server:')

GET / HTTP/1.1
Host: foo.com
Cookie: bar=42; session=12345

HTTP/1.1 200 OK
...
